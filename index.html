Deep dive

A fleet of automated Python services ingesting live WebSocket feeds from five exchanges and prediction markets, running around the clock. The code is private; the architecture is documented publicly.

  • Six-layer connection-hardening pattern — warm-up, multi-replica failover, stale-tick rejection, staggered reconnects — keeps 16 concurrent feeds live through network faults and exchange outages.

  • Event streams persisted to SQLite for offline replay, with reconciliation checks verifying zero NaN across 78,000+ time-series windows.

About
